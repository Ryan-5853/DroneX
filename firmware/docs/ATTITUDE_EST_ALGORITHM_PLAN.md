# 双正交 IMU 姿态估计算法设计计划

> 在 `Algorithm/Attitude_Est` 中实现融合两个正交 IMU 的姿态估计算法。

---

## 一、现状与目标

### 1.1 硬件现状

| 项目 | 规格 |
|------|------|
| IMU 型号 | ICM42688 × 2（六轴：陀螺 + 加速度计） |
| 安装方式 | **正交安装**（两个 IMU 敏感轴相互正交 90°） |
| 采样率 | 200 Hz（IMU 读取） |
| 解算率 | 100 Hz（姿态任务） |
| 数据单位 | gyro [rad/s]，accel [m/s²] |

### 1.2 正交双 IMU 的优势

- **冗余测量**：同一物理量（角速度、加速度）由两个正交轴系测量，可降低噪声、提高精度
- **故障检测**：两个独立 IMU 可做一致性检验，检测单 IMU 故障
- **偏置估计**：多传感器融合可更好估计陀螺/加速度计偏置

### 1.3 设计目标

1. 将两个正交 IMU 的测量统一到**机体系**（body frame）
2. 融合后输出：姿态（四元数/欧拉角）、角速度、加速度
3. 支持 IMU 校准（偏置估计）
4. 算法在 STM32H743 上实时运行（100 Hz）

## 二、算法选型

### 2.1 推荐方案：分阶段实现

| 阶段 | 算法 | 复杂度 | 适用场景 |
|------|------|--------|----------|
| **P1（首版）** | 互补滤波 + 正交融合 | 低 | 快速验证、资源受限 |
| **P2（进阶）** | 扩展卡尔曼滤波（EKF） | 中 | 精度提升、偏置在线估计 |
| **P3（可选）** | 考虑传感器位置的集中式 EKF | 高 | 两 IMU 距离较大时 |

### 2.2 算法详解

#### 方案 A：互补滤波 + 正交融合（推荐首版）

**原理**：
- 陀螺：低频漂移、高频响应好 → 用于姿态积分
- 加速度计：高频噪声大、低频可测重力方向 → 用于姿态修正
- 互补滤波：高通滤陀螺、低通滤加速度计，两者互补

**正交融合策略**：
1. **坐标系转换**：将 IMU2 的测量通过安装旋转矩阵 \( R_{2 \to 1} \) 转换到 IMU1（或机体系）
2. **数据融合**：对转换后的角速度、加速度做**加权平均**或**最优线性组合**
3. **姿态解算**：对融合后的角速度积分，用融合后的加速度做倾角修正

**优点**：实现简单、计算量小、易于调试  
**缺点**：偏置需离线校准，无在线估计

---

#### 方案 B：扩展卡尔曼滤波（EKF）

**原理**：
- 状态向量：\( x = [q, \omega, b_\omega, b_a]^T \)（四元数、角速度、陀螺偏置、加速度偏置）
- 预测：陀螺积分姿态，偏置建模为随机游走
- 更新：加速度计测量重力方向、两个 IMU 的冗余测量作为观测

**正交融合**：
- 将 IMU2 测量转换到机体系后，作为**额外观测**进入 EKF 测量方程
- 观测噪声协方差可根据传感器质量设置不同权重

**优点**：最优线性估计、可在线估计偏置、理论完备  
**缺点**：实现复杂、需调参（Q、R 矩阵）

---

#### 方案 C：集中式 Kalman 滤波（考虑传感器位置）

**适用**：两个 IMU **非共点安装**（存在位移）时。

**原理**（参考 MDPI Sensors 2018）：
- 加速度计受**离心加速度** \( \omega \times (\omega \times r) \) 和**切向加速度** \( \dot{\omega} \times r \) 影响
- 通过 Grubin 变换将各 IMU 测量统一到质心
- 状态包含各 IMU 偏置，集中式滤波同时估计

**适用条件**：两 IMU 间距 > 数毫米且角速度较大时才有显著影响。  
**本项目**：若两 IMU 紧贴或共点，可**不考虑**传感器位置，简化实现。

## 三、正交安装与坐标系

### 3.1 安装约定

设：
- **机体系** \( \mathcal{B} \)：以 IMU1 为参考，或取机体几何中心
- **IMU1 系**：与机体系对齐（或已知固定变换）
- **IMU2 系**：相对 IMU1 **正交安装**，即 \( R_{2 \to 1} \) 为 90° 旋转矩阵

**典型正交配置示例**（绕 Z 轴转 90°）：
```
IMU1: X→前, Y→右, Z→下
IMU2: X→右, Y→后, Z→下  （绕 Z 转 90°）
```
则 \( R_{2 \to 1} \) 将 IMU2 的 (x,y,z) 映射到机体系的 (y,-x,z)。

### 3.2 转换公式

设 IMU2 测量为 \( \omega_2, a_2 \)（其本体系），则机体系下：
```
ω_body = R_{2→1} · ω_2
a_body = R_{2→1} · a_2
```

**注意**：具体 \( R_{2 \to 1} \) 需根据**实际 PCB 布局与机械安装**确定，应在 `Attitude_Est` 中可配置。

```

├── Algorithm/
│   └── Attitude_Est/
│       ├── Attitude_Est.h      # 对外接口
│       ├── Attitude_Est.c      # 主逻辑：融合 + 姿态解算
│       ├── Attitude_Est_Fusion.c   # 正交融合（坐标变换 + 加权）
│       ├── Attitude_Est_Filter.c   # 互补滤波 / EKF
│       └── Attitude_Est_Config.h   # 安装矩阵、滤波参数
├── Application/
│   └── Attitude/               # 应用层：调用 Algorithm，处理校准请求
│       ├── Attitude.c
│       └── Attitude.h
└── ...
```

### 4.2 数据流

```
IMU1 原始数据 ──┐
                ├──→ 正交融合 ──→ 融合后 (ω, a) ──→ 姿态滤波 ──→ 姿态输出 (q, euler, ω)
IMU2 原始数据 ──┘
```

### 4.3 接口定义（草案）

```c
/* 输入：两个 IMU 的原始数据（机体系或需转换） */
void Attitude_Est_Update(const IMU_Data_t *imu1, const IMU_Data_t *imu2, float dt_s);

/* 输出：姿态、角速度 */
void Attitude_Est_GetQuat(float q[4]);
void Attitude_Est_GetEuler(float *roll_rad, float *pitch_rad, float *yaw_rad);
void Attitude_Est_GetAngularRate(float omega[3]);

/* 校准：设置偏置（由 Attitude 应用层调用） */
void Attitude_Est_SetBias(int imu_id, const IMU_Bias_t *bias);
```

## 五、实现步骤（P1 互补滤波 + 正交融合）

### Step 1：坐标系与融合

1. 定义 `R_imu2_to_body`（3×3 旋转矩阵），根据实际安装填写
2. 实现 `Attitude_Est_Fusion_Update(imu1, imu2, &omega_fused, &acc_fused)`
   - 将 IMU2 转换到机体系
   - 角速度：(ω1 + R·ω2) / 2 或按方差加权
   - 加速度：同理

### Step 2：互补滤波姿态解算

1. 姿态表示：四元数 q
2. 预测：\( q_{k+1} = q_k \otimes \exp(\omega \cdot dt) \)（或一阶欧拉积分）
3. 修正：用加速度计估计的倾角与当前姿态比较，生成修正角速度，注入互补滤波
4. 经典形式：`angle = α * (angle + ω*dt) + (1-α) * accel_angle`，α 为高通系数

### Step 3：与 Application 层对接

1. `Attitude_Task` 中获取 IMU1、IMU2 数据（需在 User_main 或数据总线中提供）
2. 调用 `Attitude_Est_Update(imu1, imu2, 0.01f)`
3. 将 `Attitude_Est_GetEuler` 等结果供控制或遥测使用

### Step 4：校准集成

1. `Attitude_RequestImuCali` 触发后，在静止期采集数据
2. 计算陀螺偏置（均值）、加速度计偏置（与重力对比）
3. 调用 `Attitude_Est_SetBias` 写入，后续测量中减去

## 六、参数与可调项

| 参数 | 含义 | 典型值 |
|------|------|--------|
| 互补滤波系数 α | 陀螺权重（高通） | 0.96 ~ 0.98 |
| 融合权重 | IMU1 vs IMU2 | 0.5/0.5 或按噪声方差 |
| 解算周期 dt | 姿态更新周期 | 0.01 s（100 Hz） |
| 安装矩阵 | R_imu2_to_body | 由机械确定 |

## 八、总结

| 项目 | 建议 |
|------|------|
| **首版算法** | 互补滤波 + 正交融合（加权平均） |
| **姿态表示** | 四元数（避免万向锁） |
| **融合方式** | 坐标变换后等权平均，后续可按方差加权 |
| **进阶方向** | EKF 在线偏置估计、故障检测 |
| **安装矩阵** | 必须根据实际 PCB 与机械安装标定 |

建议先实现 **P1 互补滤波 + 正交融合**，验证双 IMU 数据流与姿态输出正确后，再考虑引入 EKF 或更复杂的融合架构。
